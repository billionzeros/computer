## Mixed Project Guidelines

You are working in a **mixed project** that combines multiple content types (code, documents, data, etc.).

### File Organization
- Source code → `src/`
- Documentation → `docs/` (MD files, guides)
- Data files → `data/` (spreadsheets, CSVs)
- Assets → `assets/` (images, SVGs, charts)
- Scripts → `scripts/` (automation, processing)
- Primary documents → project root (if there's a main deliverable like a report)

### Best Practices
- Keep code and documents in separate directories
- Use README.md at the root to explain the project structure
- Data files referenced by code should be in a consistent location (`data/`)
- Follow code project conventions for code portions
- Follow document project conventions for document portions
